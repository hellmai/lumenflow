Use this page when adding or moving validation logic in LumenFlow.

## Ownership Model

Validation concerns are split by responsibility:

- `@lumenflow/core` owns reusable schema and domain validation.
- `@lumenflow/cli` owns command-level validation flow and user-facing command validation wiring.
- Runtime packages (for example `@lumenflow/runtime`) own package-specific runtime validation.
- Docs ownership for validator architecture sits in `Content: Framework Docs`.

## Validator Map

| Concern                              | Primary modules                                                                                                                                                      | Owner                        |
| ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------- |
| WU schema and normalization          | `packages/@lumenflow/core/src/wu-schema.ts`, `packages/@lumenflow/core/src/wu-schema-normalization.ts`                                                               | `Framework: Core Validation` |
| WU lifecycle semantic checks         | `packages/@lumenflow/core/src/wu-done-validation.ts`, `packages/@lumenflow/core/src/wu-preflight-validators.ts`                                                      | `Framework: Core Validation` |
| Path/dependency validation           | `packages/@lumenflow/core/src/code-path-validator.ts`, `packages/@lumenflow/core/src/docs-path-validator.ts`, `packages/@lumenflow/core/src/dependency-validator.ts` | `Framework: Core Validation` |
| CLI command validation orchestration | `packages/@lumenflow/cli/src/wu-claim-validation.ts`, `packages/@lumenflow/cli/src/wu-create-validation.ts`, `packages/@lumenflow/cli/src/wu-edit-validators.ts`     | `Framework: CLI Enforcement` |
| CLI validator composition            | `packages/@lumenflow/cli/src/shared-validators.ts`, `packages/@lumenflow/cli/src/validator-registry.ts`                                                              | `Framework: CLI Enforcement` |
| Startup environment validation       | `packages/@lumenflow/core/src/startup-env-validation.ts`                                                                                                             | `Framework: Core Validation` |
| Runtime daemon input validation      | `packages/@lumenflow/runtime/src/daemon/runtime-daemon.ts`                                                                                                           | `Operations: Runtime`        |

## Placement Rules

1. Add shared rules to `@lumenflow/core`.
2. Keep command-specific behavior in `@lumenflow/cli`.
3. Keep package-specific runtime checks in the owning runtime package.
4. Avoid duplicate rules across packages.
